paperctl — Conference paper management tool

USAGE
  paperctl <command> [--dir <path>] [--paper <name>] [args...]

COMMANDS
  start              Pull all repos (run before every work session)
  sync               Full bidirectional sync (pull + push all remotes)
  push [msg]         Commit & push repos that have changes
  pull-overleaf      Pull latest from Overleaf
  pull-upstream      Pull from upstream (fork repos only)
  check              Run format compliance checks
  validate           Static LaTeX validation (refs, cites, files, pages)
  strip              Strip \cyl{}/\tingru{} to _clean/ (originals untouched)
  lint               Writing-style lint (BAN rules)
  preflight          Submission preflight (anonymity, TODOs, conflicts)
  heatmap            Per-section change heatmap (student activity)
  init               Bootstrap repos from conference.json
  status             Show conference & paper status
  autostatus         Auto-detect paper status from section content
  pages              Extract page counts from compiled PDFs
  digest             Show recent changes (Overleaf/upstream) per paper
  report             Student activity report (compare pre/post sync state)
  dashboard          Auto-generate README dashboard from conference.json
  compile            Compile all papers (pdflatex → bibtex → pdflatex ×2)
  wordcount          Word count per section (prose only, strips LaTeX)
  patch              Apply SEARCH/REPLACE patch files to papers
  bib-check          Validate bibliography (undefined cites, duplicates)
  ref-check          Check cross-references (undefined refs, missing figs)
  overview           Unified dashboard (pages, words, git, activity)
  setup              Check environment & guide TeX Live installation
  help               Show this help

GLOBAL FLAGS
  --dir <path>       Path to conference directory (auto-detected from $PWD)
  --paper <name>     Operate on a single paper only
  --repo <name>      Alias for --paper

CROSS-PLATFORM
  paperctl runs on macOS, Linux (Debian/Ubuntu, Fedora, Arch), and Docker.
  TeX Live is required for: compile, validate --compile
  All other commands work without TeX Live.

  Quick setup:
    paperctl setup                          # check what's installed
    sudo bash setup-texlive.sh              # install TeX Live (auto-detects platform)
    docker build -t paperctl .              # build Docker image (includes TeX Live)
    docker run --rm -v ./eccv2026:/workspace paperctl compile  # compile in Docker

SETUP
  1. Create a conference directory (e.g. ~/Project/Papers/eccv2026/)
  2. Add a conference.json with conference info + paper list
  3. Run: paperctl init     (clones repos, sets up remotes)
  4. Run: paperctl start    (daily pre-session sync)

EXAMPLES
  cd ~/Project/Papers/eccv2026
  paperctl start                          # sync before working
  paperctl push "fix: abstract typo"      # commit & push changes
  paperctl push --paper elsa "fix typo"   # push single paper
  paperctl check                          # format compliance check
  paperctl status                         # overview dashboard

  # Monitor student changes
  paperctl digest                          # all papers' Overleaf changes
  paperctl digest --paper elsa             # single paper
  paperctl digest --todos-only             # just TODO/FIXME markers
  paperctl digest --output digest.md       # save to file

  # Student activity report (after `paperctl start`)
  paperctl report                         # show what changed since last sync
  paperctl report --paper fadir           # single paper
  paperctl report --update-notes          # also update conference.json notes
  paperctl report --output report.md      # save to file

  # Auto-generate dashboard
  paperctl dashboard                       # print to stdout
  paperctl dashboard --output README.md    # save to file
  paperctl dashboard --format json         # JSON output
  paperctl dashboard --status STATUS.md    # generate STATUS.md progress table

  # Sync with advanced options
  paperctl sync --parallel                 # sync all repos concurrently
  paperctl sync --auto-resolve             # auto-resolve merge conflicts (take theirs)
  paperctl sync --parallel --auto-resolve  # combine both
  paperctl sync --repo elsa               # sync single repo only

  # Auto-detect paper status from section content
  paperctl autostatus                      # show detected status for all papers
  paperctl autostatus --paper elsa         # single paper
  paperctl autostatus --update             # update conference.json (upgrade only)

  # Extract page counts from compiled PDFs
  paperctl pages                           # show page counts for all papers
  paperctl pages --paper textnav           # single paper
  paperctl pages --update                  # write page counts to conference.json

  # Pre-push validation
  paperctl validate                       # check all papers
  paperctl validate --paper textnav       # single paper
  paperctl validate --compile             # also compile locally (needs pdflatex)

  # Camera-ready preparation
  paperctl strip --paper textnav          # create _clean/ with stripped macros
  paperctl strip --paper textnav --diff   # preview changes without writing
  paperctl strip --remove-comments        # also remove comment blocks

  # Writing quality
  paperctl lint --paper textnav           # check BAN rules

  # Submission preflight
  paperctl preflight                      # full checklist (anonymity, TODOs, etc.)
  paperctl preflight --paper textnav      # single paper

  # Student activity heatmap
  paperctl heatmap                        # all papers, last 3 days
  paperctl heatmap --since "1 day ago"    # custom time range

  # Compile all papers locally
  paperctl compile                          # compile all, report page counts
  paperctl compile --paper ewm             # compile single paper
  paperctl compile --parallel              # compile concurrently
  paperctl compile --clean                 # remove aux files after

  # Word count analysis
  paperctl wordcount                        # per-section breakdown
  paperctl wordcount --summary             # totals only
  paperctl wordcount --paper textnav       # single paper

  # Apply patch files (from Claude Code review)
  paperctl patch --paper ewm patch.md       # apply SEARCH/REPLACE hunks
  paperctl patch --paper ewm --dry-run p.md # preview without modifying
  paperctl patch --paper ewm --reverse p.md # reverse a patch

  # Bibliography validation
  paperctl bib-check                        # check all papers
  paperctl bib-check --paper ewm           # single paper
  paperctl bib-check --unused              # also show unused bib entries

  # Cross-reference check
  paperctl ref-check                        # check all papers
  paperctl ref-check --paper textnav       # single paper

  # New conference
  mkdir ~/Project/Papers/neurips2025
  vim ~/Project/Papers/neurips2025/conference.json
  paperctl init --dir ~/Project/Papers/neurips2025
